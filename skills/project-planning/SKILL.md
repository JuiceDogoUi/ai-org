# Project Planning

## Estimation
- Use relative sizing (S/M/L/XL) over hours
- Compare to previously completed work
- Include uncertainty -- estimate ranges, not points
- Account for testing, review, and deployment

## Sprint Planning
1. Review velocity (average completed per sprint)
2. Select items up to velocity capacity
3. Break items into tasks
4. Identify dependencies and blockers
5. Commit to sprint goal

## Task Breakdown
- Each task should be completable in one day or less
- Include testing as explicit tasks
- Include review and integration tasks
- Flag external dependencies
