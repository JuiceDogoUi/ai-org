# UX Writing

## Principles
- **Clear over clever**: Users are trying to do something, not be entertained
- **Concise**: Use the fewest words that convey the meaning
- **Useful**: Every word should help the user take action or understand state
- **Consistent**: Same action = same label everywhere

## Button Labels
- Use verbs: "Save", "Delete", "Send", not "OK" or "Submit"
- Be specific: "Save changes" > "Save", "Delete account" > "Delete"
- Match the action to the consequence: "Remove from cart" not "Delete"

## Empty States
- Explain what goes here
- Show how to get started
- Example: "No projects yet. Create your first project to get started."

## Loading States
- Tell users what's happening: "Loading your dashboard..."
- For long operations: show progress or steps
